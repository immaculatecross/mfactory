03 / Unit of work

One feature, one branch, one PR

1. BriefForeman writes one bounded work order.
2. BuildFresh Builder implements, tests, logs, pushes.
3. GateCI and isolated review judge the head SHA.
4. MergeOnly green CI + approved audited verdict.

Any repair creates a new SHA, which invalidates the old review and reruns the gates. The Foreman allows one repair path, then records a block instead of looping indefinitely. Build playbook · Review playbook · PR #6: atomic verdict audit

05 / Observed dogfood

Mora exposed both value and friction

  1. Bootstrap was coupled.

    Define artifacts could not pass the scaffold's deliberately failing tool scripts alone, so the first feature also had to establish the toolchain.

  2. The first Builder ended unexpectedly.

    The session terminated unexpectedly after leaving usable uncommitted work. A replacement recovered from the dirty worktree because no factory timeout, heartbeat, or restart path exists.

  3. Review found a real false green.

    The first Reviewer showed that coverage measured only page.tsx and CI skipped a production build that rewrote tracked TypeScript config.

  4. The single repair path worked.

    The Builder repaired coverage scope and build idempotence, then CI reran on the new commit.

  5. The new SHA required fresh review.

    The build playbook dispatches review only after green CI; branch protection independently requires both checks before Mora PR #1 can merge.

06 / Why a cycle takes time

Verification latency is not orchestration latency

Productive gate latency

  • Fresh sessions reread durable artifacts instead of inheriting hidden context.
  • CI executes tripwires, secret scan, code gates, coverage and PR rules.
  • Independent review attempts to falsify green results.
  • Every repair gets a new SHA, fresh CI and a fresh verdict.

This time buys evidence, isolation and an auditable merge.

Avoidable orchestration friction

  • Manual worker dispatch and status polling.
  • No timeout, heartbeat or automatic crash recovery.
  • Manual readiness selection without the task graph.
  • Shared reviewer credential and no scaffold sync/version path.

This time is accidental and should be engineered out.

07 / Improvement queue

Lessons from Mora, not shipped claims

  1. L-001 Generate a Define bootstrap when a scaffold lacks one.
  2. L-002 Formalize the F-000 coupling or permit a narrow pre-toolchain docs gate.
  3. L-003 Separate reviewer credentials and enforce actor independence.
  4. L-004 Stamp factory version and provide a reviewable scaffold sync path.
  5. L-005 Add worker timeout, heartbeat, restart and dirty-worktree recovery reporting.
  6. L-006 Add scaffold-owned coverage-scope and production-build idempotence gates.
